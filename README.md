# 🏸 Badminton Manager
<img width="1367" height="821" alt="image" src="https://github.com/user-attachments/assets/4c529009-67a1-4962-b9d7-649ec1c56aa7" />

**Badminton Manager**는 배드민턴 클럽 및 동호회 모임에서 **게임 매칭, 코트 배정, 경기 기록**을 효율적으로 관리하기 위해 제작된 웹 애플리케이션입니다.

별도의 서버 구축이나 회원가입 없이, **브라우저 하나만 있으면 즉시 사용**할 수 있습니다. 운영진의 수고를 덜고, 모든 회원에게 공정한 게임 기회를 제공하는 것을 목표로 합니다.

## 주요 기능 

### 1. 선수 관리
*   **간편 등록:** 이름, 성별, 급수(A~F)만 입력하여 빠르게 등록.
*   **지각자 패널티:** 늦게 온 회원은 '현재 인원의 평균 게임 수'가 초기값으로 자동 적용되어 형평성을 유지합니다.
*   **검색 기능:** 대기자가 많을 때 이름으로 빠르게 찾을 수 있습니다.

### 2. 스마트 매칭 시스템
*   **자동 매칭:** 버튼 하나로 4명의 선수를 자동으로 선발하고 팀을 나눕니다.
*   **수동 매칭:** 운영진 권한으로 특정 4명을 직접 선택하여 매칭할 수 있습니다.
*   **선수 교체:** 이미 매칭된 선수를 클릭하여 대기 중인 다른 선수와 즉시 교체 가능합니다.
*   **종목 설정:** 코트별로 남복/여복/혼복/자율 설정을 지원합니다.

### 3. 코트 및 게임 운영
*   **2단 대기열:** [진행 중(Current)] 게임과 [다음 대기(Next)] 게임을 미리 배정하여 끊김 없는 진행이 가능합니다.
*   **실시간 타이머:** 게임이 시작되면 자동으로 시간이 측정됩니다.
*   **자동 승계:** 앞 게임이 종료되면 대기하던 다음 팀이 자동으로 입장합니다.

### 4. 데이터 관리 및 기록
*   **경기 기록:** 시작/종료 시간, 코트 번호, 참여 선수 명단이 기록됩니다.
*   **백업/복구:** 모든 데이터(선수, 코트, 기록)를 JSON 파일로 다운로드하고 복구할 수 있습니다. (기기 변경 시 유용)
*   **서버리스(Serverless):** 모든 데이터는 사용자의 브라우저(LocalStorage)에 저장됩니다.

## 매칭 알고리즘 

이 시스템은 **'공정성'** 과 **'다양한 교류'** 를 최우선 가치로 둡니다.

1.  **기회 균등 (Priority 1):** 총 게임 수(Game Count)가 가장 적은 사람을 0순위로 배정합니다.
2.  **랜덤 셔플 (Priority 2):** 게임 수가 같다면 무작위로 섞습니다. (일찍 온 사람들끼리만 계속 치는 고착화 방지)
3.  **파트너 중복 방지 (History Check):** 선발된 4명 중, 과거에 파트너를 했던 횟수가 가장 적은 조합을 찾습니다.
4.  **팀 밸런싱 (Tier Balancing):** 급수를 고려하여 최대한 대등한 경기가 되도록 팀을 나눕니다. (예: 1순위+4순위 vs 2순위+3순위)

## 시작하기

이 프로젝트는 단일 HTML 파일로 구성되어 있습니다.

1.  **실행:** `html` 파일을 다운로드하여 PC, 태블릿, 모바일의 웹 브라우저(Chrome, Safari 등)에서 엽니다. (안드로이드는 순정으로 가능하나, IOS는 별도의 html viewr 필요)
2.  **선수 등록:** 좌측 상단 `+ 선수 등록` 버튼을 눌러 회원을 추가합니다.
3.  **코트 추가:** 상단 `+ 코트` 버튼을 눌러 체육관 상황에 맞게 코트를 생성합니다.
4.  **게임 시작:** 코트의 `⚡ 자동 매칭` 버튼을 누릅니다.

## 주의사항

*   **데이터 저장:** 이 시스템은 서버가 없습니다. 브라우저 캐시를 삭제하거나 시크릿 모드를 닫으면 데이터가 사라질 수 있습니다.
*   **백업 권장:** 중요한 모임이 끝난 후나 기기를 변경할 때는 반드시 **[⚙️ 설정] -> [💾 데이터 백업]** 기능을 이용해 파일을 저장해 두세요.
